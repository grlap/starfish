Regenerate `docs/DESIGN-INDEX.md` by scanning the entire workspace.

## Step 1: Find all design documents

Glob for markdown files in:
- `*/docs/*.md` (per-crate docs directories)
- Root-level `.md` files: `bug.md`, `docs/raft.md`

Exclude: `README.md`, `CLAUDE.md`, `.claude/` directory, `docs/ARCHITECTURE.md`, `docs/DESIGN-INDEX.md` itself.

## Step 2: Classify each document

Read each markdown file and extract:

1. **Title**: First `#` heading
2. **Crate**: Infer from directory path (e.g., `starfish-core/docs/` → core)
3. **Category**: One of: Design, Feature Design, Plan, Analysis, Bug Tracker, Benchmark, Reference
4. **Status**: Look for explicit `**Status:**` header first. If absent, infer:
   - Contains "NOT IMPLEMENTED" or "Proposal" → NOT IMPLEMENTED
   - Contains phase markers with some done → PARTIAL
   - Describes existing code with references to real files → IMPLEMENTED
   - Bug tracker files → N/A (use open/resolved counts instead)
5. **Key files**: Source files referenced in the document
6. **For bug trackers**: Count items. ~~Strikethrough~~ titles = resolved. Everything else = open.

## Step 3: Generate DESIGN-INDEX.md

Write `docs/DESIGN-INDEX.md` with this structure:

```markdown
# Design Doc Index

> Auto-generated by `/sync-design-index`. Do not edit manually.

## Design Documents

| Doc | Crate | Status | Category | Key Files |
|-----|-------|--------|----------|-----------|
| [Title](relative/path.md) | crate | STATUS | Category | `file.rs` |

## Bug Trackers

| File | Crate | Open | Resolved |
|------|-------|------|----------|
| [path](relative/path.md) | crate | N | M |

## Reference Documents

| Doc | Crate | Category |
|-----|-------|----------|
| [Title](relative/path.md) | crate | Category |
```

Use relative paths from the `docs/` directory (e.g., `../starfish-core/docs/bugs.md`).
